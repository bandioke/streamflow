# StreamFlow v2.2 - Deployment Guide

## 🚀 Quick Deployment to VPS

This guide will help you deploy StreamFlow v2.2 with all new features to your VPS.

---

## 📋 Prerequisites

- Ubuntu 20.04+ VPS
- Node.js 20+
- FFmpeg installed
- PM2 (for process management)
- Domain name (optional, for HTTPS)

---

## 🔧 Step-by-Step Deployment

### 1. Update Repository on VPS

**Option A: If you already have StreamFlow installed**

```bash
# Navigate to your StreamFlow directory
cd ~/streamflow  # or wherever your installation is

# Stop the application
pm2 stop streamflow

# Backup your data (IMPORTANT!)
cp -r db ~/backup_db_$(date +%Y%m%d)
cp -r public/uploads ~/backup_uploads_$(date +%Y%m%d)
cp .env ~/.env_backup_$(date +%Y%m%d)

# Pull latest changes
git pull origin main

# Install any new dependencies (if any)
npm install

# Continue to step 2...
```

**Option B: Fresh Installation**

```bash
# Clone the repository
git clone https://github.com/bandioke/streamflow
cd streamflow

# Install dependencies
npm install

# Continue to step 2...
```

---

### 2. Configure Environment Variables

```bash
# Generate secure session secret
node generate-secret-enhanced.js
```

This will:
- ✅ Create `.env` file from `.env.example`
- ✅ Generate secure 128-character session secret
- ✅ Add `.env` to `.gitignore` if not already there

**Review and customize `.env` file:**

```bash
nano .env
```

**Recommended settings for production:**

```env
# Server Configuration
PORT=7575
NODE_ENV=production

# Session Security (already generated)
SESSION_SECRET=<auto-generated-secret>

# Enable new features
ENABLE_ANALYTICS=true
ENABLE_NETWORK_MONITORING=true
ENABLE_ADAPTIVE_BITRATE=true
AUTO_ADJUST_QUALITY=true

# Network thresholds (adjust based on your server)
BANDWIDTH_WARNING_THRESHOLD_MBPS=50
BANDWIDTH_CRITICAL_THRESHOLD_MBPS=80

# Bitrate limits (adjust based on your needs)
MIN_BITRATE_KBPS=500
MAX_BITRATE_KBPS=8000

# Analytics retention
ANALYTICS_RETENTION_DAYS=90

# Rate limiting
ENABLE_RATE_LIMITING=true
RATE_LIMIT_WINDOW_MS=900000
RATE_LIMIT_MAX_REQUESTS=100
```

---

### 3. Setup HTTPS/SSL (Recommended for Production)

**Using Let's Encrypt (Free SSL):**

```bash
# Install Certbot
sudo apt update
sudo apt install certbot

# Stop StreamFlow temporarily
pm2 stop streamflow

# Get SSL certificate
sudo certbot certonly --standalone -d yourdomain.com

# Certificates will be saved to:
# /etc/letsencrypt/live/yourdomain.com/fullchain.pem
# /etc/letsencrypt/live/yourdomain.com/privkey.pem
```

**Update `.env` for HTTPS:**

```env
ENABLE_HTTPS=true
SSL_CERT_PATH=/etc/letsencrypt/live/yourdomain.com/fullchain.pem
SSL_KEY_PATH=/etc/letsencrypt/live/yourdomain.com/privkey.pem
FORCE_HTTPS=true
```

**Set permissions for PM2 to read certificates:**

```bash
# Create a script to copy certificates
sudo nano /usr/local/bin/copy-ssl-certs.sh
```

Add this content:

```bash
#!/bin/bash
cp /etc/letsencrypt/live/yourdomain.com/fullchain.pem /home/ubuntu/streamflow/ssl/cert.pem
cp /etc/letsencrypt/live/yourdomain.com/privkey.pem /home/ubuntu/streamflow/ssl/key.pem
chown ubuntu:ubuntu /home/ubuntu/streamflow/ssl/*.pem
chmod 600 /home/ubuntu/streamflow/ssl/*.pem
```

```bash
# Make it executable
sudo chmod +x /usr/local/bin/copy-ssl-certs.sh

# Create SSL directory
mkdir -p ~/streamflow/ssl

# Run the script
sudo /usr/local/bin/copy-ssl-certs.sh

# Update .env to use copied certificates
nano .env
```

Update paths:

```env
SSL_CERT_PATH=/home/ubuntu/streamflow/ssl/cert.pem
SSL_KEY_PATH=/home/ubuntu/streamflow/ssl/key.pem
```

**Setup auto-renewal:**

```bash
# Add to crontab
sudo crontab -e
```

Add this line:

```
0 0 * * * certbot renew --quiet && /usr/local/bin/copy-ssl-certs.sh && pm2 restart streamflow
```

---

### 4. Restore Data (If Upgrading)

```bash
# Restore database
cp -r ~/backup_db_*/* ~/streamflow/db/

# Restore uploads
cp -r ~/backup_uploads_*/* ~/streamflow/public/uploads/

# Set permissions
chmod -R 755 ~/streamflow/public/uploads
chmod -R 755 ~/streamflow/db
```

---

### 5. Start Application

**Using PM2 (Recommended):**

```bash
# Start with PM2
pm2 start app.js --name streamflow

# Save PM2 configuration
pm2 save

# Setup PM2 to start on boot
pm2 startup

# Check status
pm2 status

# View logs
pm2 logs streamflow
```

**Or using npm:**

```bash
npm start
```

---

### 6. Verify Installation

**Check if application is running:**

```bash
pm2 status
```

**Test HTTP access:**

```bash
curl http://localhost:7575
```

**Test HTTPS access (if configured):**

```bash
curl https://yourdomain.com
```

**Check logs for any errors:**

```bash
pm2 logs streamflow --lines 50
```

---

### 7. Access Analytics Dashboard

1. Open browser and navigate to your StreamFlow URL
2. Login with your credentials
3. Click on "Analytics" in the sidebar
4. You should see:
   - Real-time network monitoring
   - Bandwidth usage charts
   - Performance metrics
   - Stream statistics

## 🐛 Troubleshooting

### Issue: Application won't start

**Check logs:**

```bash
pm2 logs streamflow --lines 100
```

**Common issues:**

1. **Port already in use:**
   ```bash
   sudo lsof -i :7575
   sudo kill -9 <PID>
   ```

2. **Missing SESSION_SECRET:**
   ```bash
   node generate-secret-enhanced.js
   ```

3. **Permission errors:**
   ```bash
   sudo chown -R $USER:$USER ~/streamflow
   chmod -R 755 ~/streamflow
   ```

---

### Issue: HTTPS not working

**Check SSL certificate paths:**

```bash
ls -la /etc/letsencrypt/live/yourdomain.com/
```

**Verify .env configuration:**

```bash
cat .env | grep SSL
```

**Check PM2 logs for SSL errors:**

```bash
pm2 logs streamflow | grep -i ssl
```

---

### Issue: Analytics not showing data

**Check if analytics service is enabled:**

```bash
cat .env | grep ENABLE_ANALYTICS
```

Should show: `ENABLE_ANALYTICS=true`

**Check database:**

```bash
sqlite3 db/streamflow.db "SELECT name FROM sqlite_master WHERE type='table';"
```

Should include: `stream_analytics`, `viewer_analytics`, `system_analytics`

**Restart application:**

```bash
pm2 restart streamflow
```

---

### Issue: Network monitoring not working

**Check if service is enabled:**

```bash
cat .env | grep ENABLE_NETWORK_MONITORING
```

**Check system dependencies:**

```bash
npm list systeminformation
```

**Restart application:**

```bash
pm2 restart streamflow
```

## 📊 Performance Tuning

### For High-Traffic Servers:

**Increase PM2 instances:**

```bash
pm2 delete streamflow
pm2 start app.js --name streamflow -i 2  # 2 instances
pm2 save
```

**Adjust bandwidth thresholds:**

```env
BANDWIDTH_WARNING_THRESHOLD_MBPS=100
BANDWIDTH_CRITICAL_THRESHOLD_MBPS=150
```

**Increase bitrate limits:**

```env
MIN_BITRATE_KBPS=1000
MAX_BITRATE_KBPS=12000
```

---

### For Low-Resource Servers:

**Disable analytics if not needed:**

```env
ENABLE_ANALYTICS=false
ENABLE_NETWORK_MONITORING=false
```

**Reduce network check frequency:**

```env
NETWORK_CHECK_INTERVAL_MS=10000  # 10 seconds instead of 5
```

**Reduce analytics retention:**

```env
ANALYTICS_RETENTION_DAYS=30  # Instead of 90
```
